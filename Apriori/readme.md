##### Apriori算法

###### 术语

频繁项集：经常出现在一块的物品集合

关联规则：暗示两种物品之间可能存在很强的关系

支持度：Support(A) = count(A)/count(samples) 该项集数目与数据记录的比值，支持度大于等于最小支持度为频繁项

置信度：Confidence(A->B) = Support(AB)/Support(A)，用于评价关联规则，置信度大于等于最小置信度具有强关联规则

###### 优缺点

优点：易于编码实现

缺点：多次遍历数据集，对大型数据集较慢

##### 生成频繁项集

先生成频繁1项集

之后循环生成2项集，去除非频繁项，生成3项集...直到无法生成新的项集

这里我们使用到Apriori原理，即如果某个项集是频繁的，那么它的子集也是频繁的，例如{0，1}是频繁的，那么{0}，{1}也是频繁的。反之，如果一个项集是非频繁的，那么它的超集也是非频繁的，这一点很有用。
